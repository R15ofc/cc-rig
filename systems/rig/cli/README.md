# RIG CLI

RIG CLI should be development-first.

Planned command groups:

- `rig package ...`
- `rig upload <gist-or-repo>`
- `rig install <package>`
- `rig remove <package>`
- `rig update`
- `rig dev init`
- `rig dev run`
- `rig dev publish`
- `rig trust info <package>`

Monitoring commands may remain available later, but they are not the core identity of RIG.
